# watermark_core

本目录用于实现一个面向 **Sway 1.2 / wlroots 0.6** 项目的
**离线屏幕鲁棒水印算法原型**。

当前阶段目标不是直接接入 compositor，
而是先完成一个“最小可运行闭环”：

1. 生成 payload
2. 生成消息模板 `Tm`
3. 生成同步模板 `Ta` / `Tb`
4. 组合得到总模板 `Tw`
5. 将 `Tw` 嵌入输入图像
6. 对攻击后图像执行检测与恢复
7. 输出：
   - `device_id`
   - `time_slot`
   - `crc_ok / fail`

---

# 1. 项目背景

本原型用于验证一个适合硕士论文的小型创新点：

- 将面向 screen-cam 场景的实时鲁棒水印方法迁移到 Wayland / Sway 场景
- 载荷仅保留取证所需的最小信息：
  - 固定设备标识
  - 粗粒度时间信息
- 不追求工业级完整方案
- 优先证明“思路成立、系统可落地、结果可写论文”

参考论文：

- **Real-time and screen-cam robust screen watermarking**

本原型只保留其核心骨架，不要求完整复现全部细节。

# 4. 载荷设计

当前采用固定长度 bitstream，不使用 QR。

建议载荷格式：

`payload = device_id(32) || time_slot(16) || crc16(16)`

其中：

- `device_id = 0x1A2B3C4D`
- `time_slot = floor((unix_time - T0) / 300)`

说明：

- `device_id` 固定，用于设备归因
- `time_slot` 为 5 分钟粒度的时间槽
- `crc16` 用于恢复结果校验
- 总长度约 64 bit

当前阶段不嵌入：

- IP
- user ID
- security level
- 其他长文本信息

# 5. 方法骨架

本原型尽量保留参考论文的总体结构：

1. 生成消息模板 `Tm`
2. 生成同步模板 `Ta`
3. 生成同步模板 `Tb`
4. 组合为总模板 `Tw`
5. 通过简化 JND 将 `Tw` 嵌入输入图像
6. 在检测端利用同步模板进行定位
7. 进行透视校正
8. 在 DFT 域恢复消息位
9. 解析 payload 并做 CRC 校验

---

# 6. 与参考论文相比的简化点

本项目是论文原型，不追求完整复现，当前已知简化方向包括：

1. **不使用 QR**
   - 用固定长度 bitstream 替代 QR 编码后的消息图案

2. **不嵌入长消息**
   - 只嵌入 `device_id + time_slot + crc16`

3. **优先单显示器**
   - 当前阶段不考虑多显示器同步问题

4. **优先 Python 原型**
   - 当前阶段的目标是快速验证闭环
   - 实时嵌入部分后续再迁移为 C

5. **JND 可做简化**
   - 如果完整实现参考论文的视觉模型过重，可先做简化 JND / 局部强度调制图

6. **检测器可先做轻量版**
   - 若完整双轮定位实现过重，可先实现简化版
   - 但必须保留：
     - 同步模板定位
     - 透视校正
     - 消息恢复
     - CRC 校验

Codex 在实现时如果进一步简化，必须同步更新本 README，并明确说明原因和影响。

# 7. 参数建议

当前建议初始参数如下：

- `device_id = 0x1A2B3C4D`
- `alpha = 0.35`
- `time_slot = floor((unix_time - T0) / 300)`

模板参数可优先参考论文中的思路：

- `L1 = floor(0.47 * min(M, N))`
- `L2 = floor(0.06 * min(M, N))`

消息模板半径可优先参考：

- `R1 = {132, 145, 159, 172}`

同步模板半径可优先参考：

- `R2 = {20, 24, 28}`

如果第一版无法完整复现，可先做合理近似，但必须在代码注释和 README 中记录。

# 8. 嵌入策略

建议嵌入公式：

`I_lum = bg + Tw * JND * alpha`

其中：

- `bg`：背景亮度
- `Tw`：总模板
- `JND`：简化视觉调制图
- `alpha`：嵌入强度

当前阶段目标不是做到最优视觉质量，而是：

- 保证模板能嵌入
- 保证后续可恢复
- 参数可调
- 接口清晰，便于迁移

---

# 9. 检测策略

检测端至少应包括：

1. 输入攻击后图像
2. 提取亮度或等效通道
3. 预处理 / 噪声分量估计
4. 基于同步模板定位
5. 透视校正
6. 恢复消息 bits
7. 解析 payload
8. CRC 校验

第一阶段优先支持：

- 原图嵌入后直接恢复
- perspective transform 后恢复
- JPEG 压缩后恢复

可选扩展：

- scaling
- blur
- brightness perturbation
- rotation
- 简化的拍摄失真模拟

# 15. 迁移到 C / Sway 的方向

当前建议后续这样迁移：

- `payload`
  - 可从 Python 原型迁移为 C
- `template_gen`
  - 可先保留离线预生成或初始化生成
- `embed`
  - 必须迁移为 C，接入 compositor 最终输出路径
- `detect`
  - 可继续保留为 Python 离线工具

建议的运行方式：

- payload 低频更新，例如每 5 分钟更新一次
- JND 低频更新，例如每 5 秒更新一次
- 每帧嵌入当前缓存的模板
